# Drift Policy

The Codex side moves quickly. Stability comes from tracking a narrow watched surface and checking it explicitly.

## What We Track

- a maintained human interest set in `docs/contracts/codex-app-server/interest-set.md`
- a machine-readable watched manifest in `contracts/codex-app-server/watch-manifest.json`
- snapshot copies of the watched generated TypeScript files in `contracts/codex-app-server/snapshots/`

## Check Against A Codex Source Tree

Use this when working alongside a local checkout of the rapidly evolving Codex repository.

```bash
python3 scripts/check_codex_app_server_contracts.py \
  --schema-dir /path/to/codex-rs/app-server-protocol/schema/typescript
```

Example for the local repo layout used during development:

```bash
python3 scripts/check_codex_app_server_contracts.py \
  --schema-dir /Users/jaju/github-others/openai/codex/codex-rs/app-server-protocol/schema/typescript
```

## Check Against The Installed Codex Binary

Use this when you want to compare the plugin’s watched surface with the schema generated by the `codex` binary currently on your `PATH`.

```bash
python3 scripts/check_codex_app_server_contracts.py --generate
```

## Refresh Snapshots Intentionally

Only do this when you have consciously reviewed the protocol drift and updated the plugin/docs accordingly.

```bash
python3 scripts/check_codex_app_server_contracts.py \
  --schema-dir /path/to/codex-rs/app-server-protocol/schema/typescript \
  --update
```

## Review Rule

A passing snapshot update is not enough.

When drift is detected:

1. identify which watched type changed
2. decide whether the plugin contract must change
3. update code and docs first if needed
4. only then refresh the watched snapshots

The snapshots are evidence of reviewed compatibility, not a substitute for review.
